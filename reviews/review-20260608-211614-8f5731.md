## High

1. File: `docs/superpowers/specs/2026-06-08-lantern-design.md:125`
   Severity: high
   Category: Diff
   Description: The Lantern spec still defines `ai_provider_requests` logging against the old contract: it says `session_id`, token counts, and cost live inside `metadata`, and it explicitly says no migration is needed. In the same review window, the new shared-platform spec and the platform-db foundation plan move those fields to first-class `ai_provider_requests` columns. If the old `metadata` pattern still needs mention, mark it as legacy/back-compat only and keep it out of the implementation path.
